# Blonix Branch Priority - Design Philosophy

## Overview

This document establishes **Blonix Branch design philosophy as the top priority** for the Re:MirAI project. All design decisions, documentation, and implementation should align with this philosophy.

## Core Principles

### 1. Light Theme First
- **Background:** `#f8fafc` (light gray)
- **Cards:** `#ffffff` (white)
- **Text:** `#0f172a` (dark gray) for primary, `#64748b` for secondary
- **Rationale:** Optimized for daytime use, accessibility, and broad appeal

### 2. Modern Color System
- **Primary:** Fuchsia/Pink (`#d946ef`) - Fresh, contemporary, energetic
- **Secondary:** Blue (`#3b82f6`) - Trustworthy, professional, clean
- **Rationale:** Appeals to general users, not requiring subculture knowledge

### 3. Typography
- **Font Family:** Inter + Poppins
- **Rationale:** Friendly, modern feel that's approachable to all users

### 4. Accessibility Priority
- **High Contrast:** WCAG AA compliant
- **Readability:** Optimized for all lighting conditions
- **Inclusivity:** Welcoming to users of all backgrounds

### 5. User-Centric Design
- **Clarity over Immersion:** Prioritize ease of use over gaming aesthetics
- **Professional Appeal:** Suitable for both personal and professional contexts
- **Broad Appeal:** No subculture knowledge required

## Target Users (Blonix Priority)

### Primary
- **General users (all ages)** seeking self-discovery through AI-powered personality insights
- Users who prefer bright, welcoming interfaces
- Users who value accessibility and readability
- Professional users seeking tools for self-reflection

### Secondary
- Teenagers and young adults (10s-20s) active on social media
- Users interested in personality trends (MBTI, horoscopes, etc.)
- Writers, creators, and role-players

## Use Cases (Blonix Priority)

### Primary Use Cases
1. **Daytime Use:** Light theme optimized for bright environments
2. **Professional Context:** Suitable for workplace or professional settings
3. **Accessibility:** High contrast for users with visual impairments
4. **Quick Access:** Fast, clear information hierarchy

### Secondary Use Cases
1. **Social Sharing:** Shareable persona cards optimized for social media
2. **Mobile Use:** Responsive design for mobile-first users
3. **Onboarding:** Frictionless entry for new users

## Implementation Guidelines

### When to Use Blonix Style
- **Always** for primary UI components
- **Always** for documentation and design system
- **Always** for new features and components
- **Always** for user-facing interfaces

### When to Consider Alternatives
- Only if specific use case requires dark theme (e.g., media consumption)
- Only if user explicitly requests dark theme (future feature)
